 문제

자연수 

N과 정수 

K가 주어졌을 때 이항 계수 

(NK)

를 10,007로 나눈 나머지를 구하는 프로그램을 작성하시오.


 접근

처음 이 문제를 읽어보았을 때, 이 문제로 이렇게 오래 고생할 줄은 몰랐다.

결론적으로 이 문제는 하루만에 풀지못하고, 결국 하루를 넘겨서야, 프로그래밍 언어를 바꿔서 성공해냈다.

일단, 문제의 조건이 Dynamic Programming을 이용하는 것이었다.

n,k = input().split(‘ ‘)
n = int(n)
k = int(k)
table = [0]*n
table[0] = 1
for i in range(1,n):
    table[i] = (table[i–1]*(i+1))%10007
result = int(table[n–1]/(table[k–1]*table[n–k–1]))
print(result%10007)

위의 코드가 Python으로 처음 제출했던 코드다.

나는 Dynamic Programming에서 주로 Memoization을 이용한다. (훨씬 구현이 쉬운 거 같다..)

처음에는 저장하는 대상을 N!로 설정했었다.

table이란 list를 만들어서 table[k]에 k!을 저장했다.

그리고 출력을 할 때에는 n!, (n-k)!, k!을 각각 호출해서 계산한 결과를 출력했다.

그런데 어떤 이유인지는 모르겠지만 계속 틀렸단 결과만 나왔다ㅠㅠ

게다가 몇몇 test case를 넣어봐도 계속 정답이 나와서 더 멘붕이 왔다.


이 문제에서 조금 독특한 점은 10,007로 나눈 값을 출력한다는 것이었다.

여러 수정 끝에도 답이 나오지 않아서 다른 사람들의 질문들을 읽어보았다.

질문들을 보면서 개인적으로 이해가 가지 않았던 부분은 ‘(a+b)%c’에 대한 설명이 많았다는 점?

당연히 C, Java 같은 언어들은 ‘(a+b)%c’와 같은 연산을 한다면 a+b 과정에서 이미 overflow가 발생할 것이다.

하지만 내가 아는 Python은 아닐 것이란 생각이 많이 들었다.

그리고, 나의 코드 상에서는 10,007로 나누는 과정에서 ‘(a+b)%c’와 같은 연산이 필요하지 않았다.

내가 무언가 놓치고 있는게 있었을까?


그래서 이항계수에 대한 다양한 원리들을 찾아보던 중에 한가지 문제점을 찾았다.

나는 modular 연산에서는 (a/b)%c=(a%c)/(b%c)가 성립을 할까?

내 머리로는 ‘아니다’란 생각이 들었다.

그래서 modular 연산과 나눗셈에 대해 구글링을 해보던 중에 조금 새로운 내용들을 배웠다.


바로 ‘페르마의 소정리’이다. (교양 수업 시간에 5살도 이해할 수 있다며 교수님이 설명하신 악몽의 소정리다.)

곱셈의 역원을 이용해 나머지를 정리하는 방법이다. 먼저 곱셈의 역원을 구하면,


modInv(k,mod) = {k^(mod-2)}%mod


가 된다. 이렇게 구한 곱셈의 역원을 이용해서


(n/k)%mod = {n*modInv(k, mod-2)}%mod


와 같이 정리할 수 있다고 한다. 이 역시 교양시간에 배웠는데 한 번 더 자세히 공부해 볼만한 것 같다.

그리고 이항 계수에 대해 조금 더 공부하던 중 잊고있었던 이항 계수의 성질을 찾았다!



이 공식을 보자마자 내 Memoization이 잘못 되었다는 것을 깨달았다.

Memoization해야할 것은 Factorial 연산이 아니라 Combination 연산이었던 것이다!

1차원적인 list에 저장할 값들이 아니였다. 그래서 바로 Python 코드를 포기하고 C언어로 코딩을 했다.


여담으로 말하자면 각 언어마다 다양한 장단점이 있지만 요즘 Python을 가장 자주 쓴다.

Python이 정말 여러가지 편리한 점이 있지만 나는 크게 3가지 이유로 쓴다.

첫번째로, ‘,’를 통해서 들어오는 input의 변수들을 따로 저장하기 편하다.

split(‘,’) 함수 하나만 이용하면 바로 변수들을 저장할 수 있으니깐!

두번째로 list 자료형의 유연함은 나의 생각을 코드로 구현하기 더 쉽게 만들어준다.

list 안에 tuple, list, bool 등 다양한 자료형이 들어갈 수 있다는 점이 코드 작성에 엄청난 편리함을 준다.

(하지만 오히려 멍청하게도 C, Java에서의 2차원 배열은 Python에서 항상 에러가 뜬다. 야매 Python 공부의 폐해…)

마지막으로 list(배열)의 길이가 가변적이라는 점이다.

물론 C와 같은 언어에도 malloc 같은 함수들을 통해 동적할당이 가능한 것으로 알고 있지만, 내가 잘 모른다…

아무튼 이 셋 중 하나에 포함될 경우-정말 많은 경우들이 여기에 포함된다!-나는 거의 무조건 Python을 사용한다.


아무튼 내가 Python에서 제일 약한 2차원 배열의 형태가 나왔기 때문에 이번 문제는 오랜만에 C로 작성했다.

(항상 가끔 C를 쓸 때마다 C++을 배워볼까라는 욕구가 매우 강하게 들곤 한다…!!)

#include <stdio.h>
 
int m[1001][1001] = {0};
int i,j;
 
int main() {
    int n,k;
    scanf(“%d %d”,&n, &k);
    m[1][0]=1;
    for(i=2;i<=n;i++){
        m[i][0]=m[i–1][0]*(i);
    }
    for(i=1; i<=n; i++) {
        m[i][1] = i;
    }
    for(i=2; i<=n;i++) {
        for(j=2; j<=i; j++){
            m[i][j]=(m[i–1][j]+m[i–1][j–1])%10007;
        }
    }
    printf(“%d”, m[n][k]);
    return 0;
}

문제의 조건이 1<=N<=1000이었기 때문에 1001*1001짜리 배열을 만들었다.

첫번째 for문을 통해서 nC0의 값들을 계산했다. nC0의 값은 n!이라 m[n-1[0]에서 n을 계속 곱해서 채웠다.

두번째 for문을 통해서 nC1의 값들을 계산했다. nC1의 값은 n이라 별로 어렵지 않았다.

세번째 for문을 통해서 이항계수의 성질을 이용해 나머지 값들을 계속 저장했다.

C언어 코드는 별 어려움 없이 바로 통과했다!


오랜 고민과 시행착오 끝에 문제를 풀어서 너무 뿌듯했다. 오랜만에 만난 수학적 지식은 덤!ㅎㅎ

공부를 하면서 참고했던 블로그 주소를 두개 밑에 적어놨다.

첫번째 주소가 내 고민의 발단이었던 (a/b)%c을 해결하는 ‘페르마의 소정리’에 대한 글,

두번째 주소는 아직 자세히 읽어보지는 않았지만 이항계수의 modular 연산에 관한 심도있는 글이다.

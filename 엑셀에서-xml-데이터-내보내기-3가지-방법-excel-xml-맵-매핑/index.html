[Excel 2016 도움말 요약] – 하단 자료 첨부

중요: XML 맵을 만들고 Excel의 데이터를 XML 데이터 파일로 내보낼 경우 내보낼 수 있는 행 수가 제한되어 있습니다. XML에서 Excel로 내보내면 최대 65,536개 행이 저장됩니다. 파일에 65,536개보다 많은 행이 있는 경우 다음 중 하나를 실행해야 합니다.

  • .xlsx 파일 형식을 사용하여 데이터를 내보냅니다.

  • 파일을 XML 스프레드시트(.xml)로 저장합니다. 이 경우 매핑이 손실됩니다.

  • 65,536행 다음의 행을 모두 삭제한 다음 다시 내보냅니다. 이 경우, 매핑은 유지되지만 파일 끝부분에 있는 데이터가 손실됩니다.

1. XML 맵을 만들어야 한다.

2. 65,536행 이상은 XML 데이터구조로는 불가능하다.

 1) xlsx 로 변경한다.

 2) XML 스프레드시트(.xml) 로 저장 ( 매핑 손실 됨)

 3) 65,536행 이후 행 삭제 후 다시 내보내기 ( 매핑 유지, 끝부분 데이터 손실 됨)

Excel 2016 도움말 참조

2016 Excel에서에서 XML 데이터 내보내기

중요:  본 문서는 기계 번역된 문서이므로 고지 사항을 확인하십시오. 이 문서의 영문 버전은 여기서 확인할 수 있습니다.

XML 데이터를 가져와서 워크시트의 셀에 매핑하고 데이터를 변경한 후 XML 파일로 내보내거나 저장할 수 있습니다.

중요: XML 맵을 만들고 Excel의 데이터를 XML 데이터 파일로 내보낼 경우 내보낼 수 있는 행 수가 제한되어 있습니다. XML에서 Excel로 내보내면 최대 65,536개 행이 저장됩니다. 파일에 65,536개보다 많은 행이 있는 경우 다음 중 하나를 실행해야 합니다.

  • .xlsx 파일 형식을 사용하여 데이터를 내보냅니다.

  • 파일을 XML 스프레드시트(.xml)로 저장합니다. 이 경우 매핑이 손실됩니다.

  • 65,536행 다음의 행을 모두 삭제한 다음 다시 내보냅니다. 이 경우, 매핑은 유지되지만 파일 끝부분에 있는 데이터가 손실됩니다.

65,536개 이하의 행이 포함된 워크시트에서 XML데이터 내보내기

  1. 개발 도구 탭의 XML 그룹에서 내보내기를 클릭합니다.

    개발 도구 탭이 표시되지 않을 경우 개발 도구 탭 표시를 참조하세요.

  2. 작은 XML 내보내기 대화 상자가 나타나면 사용할 XML 맵을 클릭한 다음 확인을 클릭합니다.

    이 대화 상자는 XML 표가 선택되지 않았고 통합 문서에 여러 XML 맵이 포함된 경우에만 표시됩니다.

  3. 큰 XML 내보내기 대화 상자의 파일 이름 상자에 XML 데이터 파일의 이름을 입력합니다.

  4. 내보내기를 클릭합니다.

    참고: 워크시트에 XML 맵의 XML 요소 이름과 다른 제목 또는 레이블이 있는 경우 워크시트에서 XML 데이터를 내보내거나 저장하면 XML 요소 이름이 사용됩니다.

65,536개보다 많은 행이 포함된 워크시트에서 XML데이터 내보내기

  1. 파일의 전체 행 수에서 65,537을 뺍니다.

    그 결과 남은 행 수는 x개입니다.

  2. Excel 워크시트 시작 부분에서 x개의 행이 삭제됩니다.

  3. 워크시트를 XML 데이터 파일로 내보냅니다.

  4. 닫기를 클릭하고 Excel 워크시트를 다시 엽니다.

  5. 총 x개 다음의 모든 행을 삭제한 다음 XML 데이터 파일로 내보냅니다.

    그러면 나머지 행만 남습니다. 이제 이 XML 내보내기 파일 2개를 결합해서 원래 워크시트의 복제본을 만들 수 있습니다.

매핑된 셀의 XML 데이터를 XML 데이터 파일로 저장

전 버전의 XML 기능과 호환성을 유지하기 위해 내보내기 명령을 사용하지 않고 파일을 XML 데이터 파일(.xml)로 저장할 수 있습니다.

  1. Ctrl+S를 눌러 파일을 저장합니다.

    이 단계를 사용하면 통합 문서를 XML 데이터 파일로 저장할 때 통합 문서의 변경 내용이 소실되지 않습니다.

  2. 파일 > 다른 이름으로 저장을 클릭하고 파일을 저장할 위치를 선택합니다.

  3. 파일 이름 상자에 XML 데이터 파일 이름을 입력합니다.

  4. 파일 형식 목록에서 XML 데이터를 클릭한 다음, 저장을 클릭합니다.

  5. 파일을 XML 데이터로 저장하는 경우 기능이 소실될 수 있다는 메시지가 나타나면 계속을 클릭합니다.

  6. 작은 XML 내보내기 대화 상자가 나타나면 사용할 XML 맵을 클릭한 다음 확인을 클릭합니다.

    이 대화 상자는 XML 표가 선택되지 않았고 통합 문서에 여러 XML 맵이 포함된 경우에만 표시됩니다.

  7. 큰 XML 내보내기 대화 상자의 파일 이름 상자에 XML 데이터 파일의 이름을 입력합니다.

  8. 내보내기를 클릭합니다.

참고: 워크시트에 XML 맵의 XML 요소 이름과 다른 제목 또는 레이블이 있는 경우 워크시트에서 XML 데이터를 내보내거나 저장하면 XML 요소 이름이 사용됩니다.

XML 데이터를 내보낼 때 발생할 수 있는 일반적인 문제

XML 데이터를 내보낼 때 아래와 같은 메시지가 나타날 수 있습니다.

XML 맵을 내보낼 수 있지만 일부 필수 요소가 매핑되지 않습니다

이 메시지가 표시되는 원인으로는 여러 가지가 있습니다.

  • XML 표에 매핑되지 않은 한 개 이상의 필수 요소가 이 XML 표와 연결된 XML 맵에 있습니다.

    XML 원본 작업창의 계층적 요소 목록에서 각 요소의 왼쪽에 관련 아이콘이 표시됩니다. 필수 요소의 경우에는 해당 아이콘의 오른쪽 위 모서리에 빨간색 별표가 표시됩니다. 필수 요소를 매핑하려면 워크시트에서 이를 표시할 위치로 해당 요소를 끕니다.

  • 요소가 재귀 구조입니다.

    재귀 구조의 일반적인 예로는 동일한 XML 요소가 여러 수준으로 중첩된 직원 및 관리자 계층 구조를 들 수 있습니다. XML 원본 작업창에서 모든 요소를 매핑하더라도 Excel에서 지원하는 재귀 구조는 한 수준까지이므로 요소 전체를 매핑할 수 없습니다.

  • XML 표에 혼합된 콘텐츠가 포함되어 있습니다.

    혼합된 콘텐츠는 요소에 자식 요소가 있고 자식 요소 외부에 간단한 텍스트가 있는 경우 발생합니다. 굵은 글꼴 태그 등의 서식 태그를 사용하여 요소 내의 데이터를 표시하는 경우를 예로 들 수 있습니다. Excel에서 자식 요소를 지원하는 경우 자식 요소를 표시할 수 있더라도 데이터를 가져오면 텍스트 내용이 소실되고 데이터를 내보내면 텍스트 내용을 사용할 수 없으므로 이러한 자식 요소는 상호 전환될 수 없습니다.

통합 문서의 XML 맵은 내보낼 수 없음

매핑된 요소와 다른 요소의 관계를 유지할 수 없는 경우 XML 맵을 내보낼 수 없습니다. 이 관계가 유지되지 않는 이유는 다음과 같습니다.

  • 매핑된 요소의 스키마 정의가 다음 특성의 시퀀스에 포함되어 있습니다.

    • maxoccurs 특성이 1이 아닙니다.

    • 시퀀스에 직계 자식 요소가 하나 이상 정의되어 있거나 직계 자식으로 다른 구성 요소가 포함되어 있습니다.

  • 같은 반복 부모 요소의 반복되지 않는 형제 요소들이 다른 XML 표에 매핑되어 있습니다.

  • 여러 반복 요소가 XML 표에 매핑되어 있고 조상 요소에서 반복을 정의하지 않았습니다.

  • 다른 부모의 자식 요소들이 같은 XML 표에 매핑되어 있습니다.

또는 XML 맵에 다음 중 하나에 해당하는 XML 스키마 구문이 포함되어 있어서 XML 맵을 내보내지 못할 수도 있습니다.

  • 목록의 목록    한 항목 목록에 다른 항목 목록이 들어 있습니다.

  • 비정규화된 데이터    XML 표에는 maxoccurs 특성을 1로 설정하여 한 번만 발생하도록 스키마에 정의된 요소가 있습니다. 이러한 요소를 XML 표에 추가하면 Excel에서 요소의 여러 인스턴스로 해당 표의 열이 채워집니다.

  • 선택    매핑된 요소가 <choice> 스키마 구문의 일부입니다.

통합 문서에 매핑된 범위가 없어 Excel에서 저장할 수 없음

하나 이상의 매핑된 범위를 만들지 않으면 Excel에서 통합 문서를 XML 데이터 파일로 저장할 수 없습니다.

통합 문서에 XML 맵을 추가했으면 다음 단계에 따라 워크시트에 XML 요소를 매핑합니다.

  1. 개발 도구 탭의 XML 그룹에서 원본을 클릭합니다.

    개발 도구 탭이 표시되지 않을 경우 개발 도구 탭 표시를 참조하세요.

  2. XML 원본 작업창에서 워크시트로 매핑하려는 요소를 선택합니다.

    인접하지 않은 여러 요소를 선택하려면 요소 하나를 클릭하고 Ctrl 키를 누른 채로 다른 요소를 클릭합니다.

  3. 워크시트에서 표시할 위치로 선택한 요소를 끕니다.

통합 문서에 XML 맵을 추가하지 않았거나 데이터가 XML 표에 포함되어 있지 않으면 통합 문서를 Excel 매크로 사용 통합 문서 파일 형식(.xlsm)으로 저장합니다.

개발 도구 탭 표시

  1. 파일을 클릭한 다음 옵션을 클릭합니다.

  2. 리본 사용자 지정 범주를 클릭합니다.

  3. 기본 탭에서 개발 도구 확인란을 선택한 다음 확인을 클릭합니다.

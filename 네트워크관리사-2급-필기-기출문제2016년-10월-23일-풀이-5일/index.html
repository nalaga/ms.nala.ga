네트워크관리사 2급 필기 기출문제 2016년 10월 23일자 풀이 마지막 날이다. 아마 이번 포스팅으로 네트워크관리사 필기 기출문제에 대한 포스팅은 끝이 될 듯 하다. 겨우 3회 정도의 분량만 풀이한 것이라 시험 합격에 얼마나 도움이 될지는 모르겠다만 그래도 이렇게 꾸준히 하려고 노력이라도 한 내 자신에게 심심한 경의를 표한다. 포스팅일 기준(2017년 10월 21일)으로 내일이 시험일(2017년 10월 22일)이다. 시험 전날 저녁에 이렇게 팔자좋게 10문제 문제 풀이나 하고 있다니 어이가 없기는 하지만 문제풀이 이후에 기출문제를 조금 더 풀 의욕이 있다면 한번 풀어볼 예정이다.
-관련글
2017/10/23 – 네트워크관리사 2급 필기 기출문제(2016년 10월 23일)풀이 4일차

네트워크관리사 2급 필기 기출문제(2016년 10월 23일) 풀이 5일차

[해설] Windows Server 2008 R2 VPN(Virtual Private Networks)
인터넷망을 전용선처럼 사용할 수 있도록 특수 통신체계와 암호화기법을 제공하는 서비스이다. 기업 본사와 지사 또는 지사 간에 전용망을 설치한 것과 같은 효과를 거둘 수 있으며 기존 사설망의 고비용 부담을 해소하기 위해 사용한다.

다. VPN 접속은 한번 접속할 때마다 VPN 연결을 해야한다.

[정답] 다

[해설] Windwos Server 2008 R2 원격 데스크톱 서비스
원격 데스크톱은 해당 컴퓨터에 대해 2개의 동시 원격 연결만 지원합니다. 이러한 연결에는 RDS CAL(원격 데스크톱 서비스  클라이언트 액세스 라이센스)이 필요하지 않다. 터미널 라이센스를 구입하면 동시 원격 연결 수를 추가적으로 늘릴 수 있다고 한다.

[정답] 가

[해설] Windows Server 2008 R2 DHCP 서버
가. 특정 호스트에게 항상 같은 IP 주소를 부여하려면 그 호스트에 대한 MAC주소가 필요하다.
다. 클라이언트가 도메인 이름 조회를 할 수 있게 하려면 DNS 서버를 지정한다.
라. DHCPv6 클라이언트가 DHCP 서버로부터 IPv6 주소를 얻기 원한다면 상태 저장 모드를 사용한다.

[정답] 나

[해설] Windows Server 2008 R2 Hyper-V 스냅숏
컴퓨터에서 실행 중인 프로그램의 상태를 알아보기 위해 어느 한 순간 주기억 장치나 하드웨어 레지스터, 상태 표시기 등의 모든 내용을 포함한 메모리의 현재 상태를 저장한 것이다. 시스템이 고장으로 정지됐을 때 복구를 위해 주기적으로 저장된다. 간단한 백업 및 롤백 용도로는 사용 가능하나 스냅 숏만드로 시스템 복구가 불가능하므로 주로 테스트 할 때 간편하게 이용할 수 있는 기능이라고 보면 된다.

[정답] 다

[해설] Windows Server 2008 R2 롤백 템플릿
가. 파일시스템과 레지스트리를 이전 상태롤 되돌리는 것은 시스템 복원으로 가능하다.

[정답] 가

[해설] 라우터(Router)
서로 다른 네트워크를 연결해주는 장치이다. 집에서 흔히 쓰는 공유기가 라우터라고 보면 된다.

가. 논리적인 주소인 IP 주소를 참조한다.
나. 패킷의 경로를 설정해주는 역할을 한다.
라. 라우터는 3계층 = 네트워크 계층(물 데 네 전 세 표 응) 장비이다.

[정답] 라

[해설] 리피터(Repeater)
디지털 방식의 통신선로에서 신호를 전송할때 전송하는 거리가 멀어지면 신호가 감쇠하는 성질이 있다. 이 때 감쇠된 전송신호를 새롭게 재생하여 다시 전달하는 재생중계장치를 리피터라고 한다.

자주 출제되는 문제이다. 눈 여겨 보도록 한다.

[정답] 라

[해설] 전송 매체
꼬임선 : 가는 전선 여러 가닥을 꼬아서 만든 전선 또는 인터넷 LAN선 같은 통신선에서 서로의 신호간섭을 줄이고자 여러가닥의 가는 개별 신호선을 꼬아서 한 묶음으로 만든 통신선이다.
동축케이블 : 방송이나 유선 TV에서 많이 쓰던 방식으로 중앙의 구리선에 절연체를 싸고 가는 선을 눌러싼 후 다시 피복을 입힌 형태로 되어 있다. 인터넷 선으로도 쓰였었다.
광케이블 : 빛이 이동할 수 있는 광섬유를 뭉쳐서 만든 굵은 통신선으로 적은 부피로 대량의 정보를 빠르고 안정적으로 전송할 수 있기 때문에 기간 통신 사업자의 전화나 인터넷 망이 거의 관케이블로 연결되어 있다.

광섬유나 광케이블에 대한 문제가 자주 출제되는 것 같다. 꼭 기억해두자.

[정답] 다

[해설] 라우팅 프로토콜 종류
OSPF, IS-IS, IGRP, EIGRP

IPX(Internetwork Packet Exchange) : IPX/SPX 프로토콜 스택 안에 있는 OSI 모델 네트워크 계층 프로토콜이다. 스타크래프트 할 때 배틀넷 안되면 주변 친구들이랑 이것으로 많이 했었던 기억이 난다. 아무는 라우팅 프로토콜이 아닌 인터네트워킹 프로토콜이라는 것을 기억하자.

[정답] 다

[해설] RAID(Redundant Array of Independent Disks, 복수 배열 독립 디스크)
여러 개의 물리 디스크를 하나의 논리적 디스크로 인식 작동하는 기술이다.

RAID 0 : 스트라이핑 저장방식으로 연속된 데이터를 여러 디스크에 나눠서 저장한다. 최소 2개의 하드디스크가 필요하고 입출력 작업이 모든 디스크에서 동시에 진행된다. 저장과 읽기 속도가 가장 빠르지만 하나의 디스크라도 고장나면 전체 사용불가하다.

RAID 1 : 미러링 방식으로 하나의 디스크에 저장하면 다른 디스크에도 동일 내용 백업 저장된다. 데이터 저장시 2배의 용량이 필요하며 결합허용을 제공하지만 공간 효율성이 떨어진다.

RAID 2 : 스트라이핑 방식으로 기록용과 데이터 복구용 디스크를 별도로 제공한다. 해밍코드를 사용하며 디스크의 사용 효율성이 낮다. 오든 SCSI 디스크에 에러검출기능을 탑재하고 있어 실제로는 사용되지 않는다.

RAID 3 : 스트라이핑 방식으로 오류 검출을 위해 패리티 방식으로 사용한다. 패리티 정보 저장을 위해 최소 3개 이상의 하드디스크가 필요하며 데이터 복구는 패리티 저장 디스크에 기록된 정보의 XOR을 연산하여 수행한다. 대형 레코드를 많이 사용하는 단일 사용자 시스템에 적합하다.

RAID 4 : RAID 3과 유사하면 2개 이상의 디스크와 전용 패리티 디스크를 사용한다. RAID 3은 바이트 단위로 저장하지만 RAID 4는 Block(sector) 단위로 저장한다.

RAID 5 : 스트라이핑 방식으로 디스크마다 패리티 정보를 가지고 있어 병목현상을 줄일 수 있다. 실무에 많이 쓰이며 섹터 단위로 저장한다. 쓰기 작업이 많지 않은 다중 사용자 시스템에 적합하다.

이해하기에는 꽤나 어려운 문제인 것 같다. 이해가 되지 않으면 정답을 외우도록 한다.

[정답] 라

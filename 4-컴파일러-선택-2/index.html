4. 컴파일러 선택

  • By admin
  • on 1월 15, 2018
  • in 미분류

3줄 요약

  1. Dev C++는 GNU 라이센스하 오픈소스 기반이어서 종류가 몇가지 있습니다.
  2. 특히 USB같은 간단한 저장매체에 설치없이 넣고 쓸수있는 포터블 버전도 있습니다.
  3. Visual Studio 2010 SP1 (서비스팩)를 설치하면 해결됩니다.

이제 본격적으로 C언어를 정리하겠습니다.

본 내용은 혹시나 저의 정리 내용을 보시고 공부하고싶으신 분들을 위한 포스팅 입니다.

 

일단 당연한 이야기로 C언어를 실습하려면 문법에 맞춰 코드를 작성하고 이를 컴파일하여 최종적으로 결과물을 보기위하여 환경을 만드셔야 하겠죠?

 

컴파일러는 용어정리에서 보셨듯, 프로그래밍 언어의 코드를  해당 플랫폼에 맞게 기계어로 번역하는 소프트웨어를 말합니다. C언어의 컴파일러에는 종류가 여러가지 있으나, 대부분의 C언어 컴파일러는 C++ 컴파일러와 함께 지원합니다.

 

요즘 개발환경은 컴파일러만 제공하는 경우는 없고, 소스코드 편집기, 디버거, 컴파일러, 링커, 리소스편집기 등등과 함께 여러가지 개발에 도움을 주는 유틸리티를 묶어 통합개발환경 (IDE)형태로 출시 되고 있습니다.

 

윈도우 환경에서의 대표적인 IDE로

 

Visual C++ ( Visual Studio에 포함되어 있습니다. MSVC라고 부르기도 합니다.
Boland C++ Builder ( Borland Developer Studio, Codegear RAD Studio에 포함되어 있습니다. )

Dev C++ ( Dev C++의 컴파일러는 오픈소스 프로젝트의 GCC 컴파일러가 들어있습니다. ) 

 

등이 있습니다.

( 사실 Boland C++은 저도 써본적은 없습니다.. ) 

 

Dev C++는 GNU 라이센스하 오픈소스 기반이어서 종류가 몇가지 있습니다.

특히 USB같은 간단한 저장매체에 설치없이 넣고 쓸수있는 포터블 버전도 있습니다.

잘 찾아보시기 바랍니다…?

 

 

사실 윈도우 환경에서 개발한다면  Visual Studio만큼 편한 IDE도 없습니다. ( 오오 마솔루야 )

윈도우가 성공한 이유중 하나 이기도 합니다. 편리한 디버거와 프로젝트 관리, 자동완성기능등등 말이죠..

물론 문법공부 수준에서는 Dev C++도 좋은 컴파일러 임에는 확실 하지만,

Visual Studio를 쓰시길 추천 합니다. 적어도 2008버전 이상이 좋습니다.

그이하 버전에서는 지원하지 않는 문법들이 있을 수 있기 때문입니다.

( 아직도 돈이 없는 학교나 회사는 Visual Studio 6.0버전을 쓴다 카더라.. )

 

 

참고

1. Visual Studio 2012버전 이상부터는 윈도우7 서비스팩1 이상에서만 설치 됩니다.

( 영원히 곧통받는 비스타…  )

 

 

2. Visual Studio 2010은 명백한 버그로 COFF변환 과정에서 링크에러로 빌드가 안되는 경우도 있는데

Visual Studio 2010 SP1 (서비스팩)를 설치하면 해결됩니다.

 

3. Visual Studio Express 에디션이 있는데 라이센스는 무료이며, 몇가지 기능이 제거되어 있습니다.

 

4. Dreamspark 라는 마이크로소프트에서 운영하는 사이트에 가셔서 가입후, 학생임을 증명하면

Visual Studio 정품을 비롯한 각종 프로그램의 다운로드와 라이센스를 무료로 제공합니다.

 

5. Visual Studio 2013버전은 Community 에디션을 출시하여 Express를 대체합니다.

Visual Studio 2013 Community 는 기존 정품의 기능을 대부분 사용가능하고 라이센스는 

비상업용 이거나, 5인이하 스타트업 사업장은 무료라고 합니다!

alter table emp_test
    add (hire_date date);


사원이 입사했을떄, 사원의 입사일자는 그걸 등록하는날이 입사일자가 되는게 대부분이니까 오늘날짜가 디폴트가 되도록  지정해보자
컬럼은 이미있다. 이 컬럼의 구조를 변경하는것 (근데 디폴트 값을 변경하는것임)


    alter table emp_test
    modify (hire_date default sysdate);


이제 값 넣어보자

    insert into emp_test
    values(102, ‘심청’,60,default, default);

  insert into emp_test(emp_id, name, dept_id)
    values(103, ‘임꺽정’,70);

이렇게 걍 안넣어도 디폴트값 입력됨


근데 만약 맘이 변해서 입사일자가 꼭 있어야한다! 로 바꾸고싶다.

alter table emp_test
    modify (hire_date not null);

근데 이미 null 값인 hire date 가 있기때문에 오류남


 emp_test 테이블의 hire_date 컬럼에 
    데이터값이 없는 행에 대해 오늘날짜로 변경해보자
    update emp_test
    set hire_date=default
    where hire_date is null;
          *=null 이 아님!! is null 임!!!!

null값이었던 홍길동과 전우치의 hire date가 변경되었다.

* constraint는 컬럼에 대해 지정하면 여러개의 컬럼에 대해 지정할 수 없기 때문에,

복합키 지정할 때는 반드시 테이블 아래에 지정해야함

하나의 테이블에 여러개 유니크 가능

하나의 테이블에 여러개의 foreign 지정가능

그런데 primary key 는 하나의 테이블에 하나만 지정가능

이말은 하나의 컬럼에만 지정할수있다는게 아니라 pk에 해당하는 컨스트레인트가 하나여야 한다는말

Wired LANs: Ethernet

 

IEEE STANDARDS

 

//오후 1:25 2014-10-20

 

1985년, IEEE 컴퓨터 협회는 다양한 제조업체의 장치들 사이의 상호 통신이 가능하도록 하는 표준 설정 Project 802라는 프로젝트를 시작했습니다. Project 802는 OSI나 인터넷 모델의 일부를 대체하는 것이 아닌 물리계층과 데이터 링크 계층의 주요한 LAN 프로토콜을 지정하여 함수화 하는 방법입니다.

 

표준은 미국 국가 표준 연구소(ANSI)에서 적용하였고, 1987년 국제 표준화 기구(ISO)또한 ISO 8802로 국제표준으로 승인했습니다.

 

802표준과 전통적인 OSI모델의 관계는 다음 그림에서 보여줍니다. IEEE는 데이터링크 계층에서 논리 연결 제어(LLC)와 미디어 접근 제어(MAC) 두 개의 서브계층으로 세분화 됩니다. 또 IEEE는 다른 LAN프로토콜에 대한 표준으로 여러 개의 물리계층도 만들었습니다.

 

IEEE standardfor LANs

Data Link Layer

앞서 언급한대로, IEEE표준에서의 데이터 링크 계층은 LLC와 MAC으로 나누어집니다.

 


Logical Link Control (LLC)

LLC는 모든 IEEE 표준을 위한 하나의 싱글 데이터 링크 컨트롤 프로토콜을 제공합니다. 같은 방식으로, LLC는 서로 다른 LAN프로토콜을 위해 미디어 접근 컨트롤(MAC) 서브 계층을 제공합니다. LLC 프로토콜은 MAC서브 계층을 투명하게 만들 수 있기 때문에 상호 서로 다른 LAN 사이의 통신이 가능하게 합니다.

LLC 프레임은 HDLC와 비슷한 프로토콜 데이터 유닛(PDU)을 정의합니다. 헤더는 HDLC의 컨트롤 필드 같은 제어필드를 포함하며 흐름과 오류제어에 사용됩니다. 서로 다른 두 헤더 필드는 LLC를 사용하는 상위계층 프로토콜의 송신지와 목적지를 정의하며, 이 필드를 목적지 서비스 접근 포인트(DSAP)와 송신지 서비스 접근 포인트(SSAP)라고 합니다.

 

LLC는 상위 계층 프로토콜의 흐름제어 및 오류제어를 제공합니다. 예로, LAN망 또는 몇몇의 LAN이 분리된 시스템에 될 때, LLC는 어플리케이션 계층 프로토콜에서 흐름이나 오류제어에 사용될 것입니다. 그러나 보통 상위 계층 프로토콜은 LLC 서비스에 사용되지 않을 것입니다.

 

Media Access Control (MAC)

IEEE Project 802는 미디어 접근 컨트롤(MAC)이라고 불리는 특정 LAN에 각각 접근 하는 방법을 정의한 서브계층에 의해 생성되었습니다. 예로는 토큰링과 토큰 버스에서의 이더넷 LAN. 토큰 패스에서의 CSMA/CD로의 미디어 접근 방법이 있습니다. 이전에 언급한 것과 같이 섹션 프레임 기능의 일부는 MAC계층에서 처리됩니다.

 

LLC계층과는 달리 MAC계층은 각각의 접근방법과 해당 LAN 프로토콜의 프레임 포맷에 대하여 정의하는 모델의 수까지 포함합니다.


Physical Layer

물리 계층은 구현과 유형에 따라 사용됩니다. IEEE는 각각의 LAN 구현에 따른 세부 특정사항을 정의합니다. 예를 들어, 표준 이더넷에 대해 하나의 MAC 서브 계층이 있지만, 각각 다른 이더넷 구현을 위한 물리계층이 있음을 볼 수 있습니다.






STANDARD ETHERNET

원조 이더넷은 1976년 Xerox’s Palo Research Center(PARC)에서 정의되었습니다.

 

MAC Sublayer

표준 이더넷은 MAC 서브계층을 적용한 접근 방식이 사용되었습니다. 그것은 상위계층과 물리계층에 전달하는 프레임 데이터를 수신하거나 적용하였습니다.

 

이더넷 프레임은 SFD, DA, SA, 프로토콜 데이터 유닛(PDU)의 길이나 유형, 상위계층의 데이터와 CRC. 일곱 개의 필드로 되어있습니다. 이더넷은 프레임 수신응답을 위한 모든 메커니즘을 제공하지는 않습니다. 따라서 응답은 상위 계층에 구현되어야 합니다. MAC 프레임과 같은 포맷은 다음과 같습니다.

 

1. 서론.

802.3 프레임의 첫 번째 필드는 0과 1을 교체할 프레임이 들어왔는지 시스템에게 알리고 그 입력에 대한 타이밍을 동기화 할 7byre(56bit)를 포함합니다. 패턴은 오직 한 개의 타이밍을 더한 경고만을 제공합니다.

 

56 bit 패턴은 프레임이 시작할 때 스테이션이 일부비트를 놓치지 않을 수 있도록 합니다. 문제는 실제로 물리계층에 추가되거나 공식적인 프레임의 일부가 아닐 경우입니다.

 

2. 시작 프레임 구분(SFD). 두 번째 필드는 신호 프레임이 시작됩니다.

SFD는 경고 스테이션이나 동기화를 위한 마지막 기회의 스테이션입니다. 마지막 2bit는 11과 다음필드의 경고수신기 필드 목적지 주소입니다.

3. 목적지 주소(DA).

DA 필드는 6Byte와 목적지 스테이션이나 패킷을 받을 스테이션의 물리적 주소를 포함합니다.

 

4. 송신지 주소(SA).

SA 필드 또한 6Byte와 패킷이 보내지는 물리 주소를 포함합니다.

 

5. 길이의 형식.

이 필드는 형식 필드나 길이 필드를 정의합니다. 기존 이더넷은 이 필드를 상위계층 프로토콜에 사용되는 MAC프레임을 정의하는 형식필드로 사용하였습니다.

 

6. 데이터.

상위계층프로토콜의 캡슐화 된 데이터를 운반합니다. 이것은 최소 46에서 최대 1500byte입니다.

 

7. CRC.

마지막 필드는 오류검출 정보를 포함합니다.

 

Frame Length

이더넷은 최소와 최대 길이 모두 제한사항이 부여됩니다.

최소 길이의 제한은 CSMA/CD의 올바른 작동을 위해 필요합니다. 하나의 이더넷 프레임은 최소 512bit(64Byte)가 필요합니다. 이 길이의 일부는 헤더와 트레일러입니다. 만약 헤더와 테리일러 길이로 18byte라고 가정한다면, 송신지 주소로 6Byte + 목적지 주소 6Byte + 길이나 형식정의 2Byte + CRC 4Byte가 되고 상위 계층으로부터 온 데이터의 최소 길이는

64 – 18로 46Byte가 됩니다. 만약 상위계층 패킷이 46Byte보다 작다면 패딩은 이 차이를

보충하기 위해 추가할 것입니다.

 

프레임에 관하여 정의된 최대 길이 표준은 1518 byte입니다. 만약 헤더나 트레일러에 18byte가 부족하다면, 페이로드의 최대 길이는 1500 byte가 될 것 입니다. 최대 길이 제한은 두 개의 원천적인 이유가 있습니다. 첫째로 이더넷을 설계함에 있어 메모리는 매우 비쌉니다.

여기서 최대 길이 제한은 버퍼의 크기를 감소시키는데 도와줍니다. 두 번째. 최대 길이 제한은 한 스테이션이 공유 매체를 독점하여 다른 스테이션의 데이터 전송을 막는 것을 방지해 줍니다.




Addressing


이더넷 네트워크(PC, 워크스테이션, 프린터와 같은)의 각각의 스테이션은 네트워크 인터페이스 카드(NIC)가 있습니다. NIC는 내부 스테이션과 맞는 스테이션을 제공하며 보편적으로, 바이트 사이의 콜론(:)과 함께 16진수로 된 6Byte/48Bit길이의 이더넷 주소를 가집니다.

 

– Ethernet address in hexadecimal notation

06:01:02:01:2C:4B

6 bytes =12 hex digits =48 bits

Physical Layer

 

표준 이더넷은 여러 물리계층 구현을 정의합니다.

 

Encoding and Decoding

 

모든 표준 구현은 디지털 신호(대역폭) 10Mbps를 사용합니다. 송신자에게 데이터는 맨체스터를 사용하여 디지털 신호로 변환되어 사용됩니다. 수신기에서 수신신호는 맨체스터로 해석하고 데이터로 디코딩되어집니다.

 

맨체스터 인코딩은 셀프 동기화, 비트단위의 전이가 있습니다.



10Base5: Thick Ethernet

Thick Ethernet이라는 이름은 손으로 구부리기에 너무 딱딱하고 대략 정원에서 사용하는 호수 크기의 케이블 사이즈에서 유래되었습니다. 10Base5는 두꺼운 동축케이블 탭으로 연결된 외부 트랜시버(송/수신기)와 버스 토플로지를 최초로 사용한 사양의 이더넷입니다.

 

트랜시버는 송수신 및 충돌감지에 대한 책임이 있습니다. 트랜시버는 송수신을 하기위한 경로를 제공하고 별도의 케이블을 통해 트랜시버 스테이션에 접속됩니다. 이것은 오직 동축 케이블 안에서만 충돌이 발생할 수 있음을 의미합니다.

 

동축케이블의 최대길이는 500m를 초과하면 안되며 그렇지 않으면 신호의 과도한 저하를 야기 시킬 수 있습니다. 만약 500m보다 더 필요할 경우 5세그먼트까지 각각에 리피터를 연결하여 사용 할 수 있습니다.

 

 

10Base2: Thin Ethernet

10Base2도 마찬가지로 버스 토플로지를 사용하지만, 케이블은 더 얇고 더 유연합니다. 케이블을 구부릴 수 있어 스테이션에 매우 가까이 지날 수 있습니다. 이 경우, 트랜스시버 (송수신기)는 보통 스테이션 내에 설치하는 네트워크 인터페이스 카드(NIC)의 일부가 됩니다.

얇은 동축케이블에서 충돌이 발생 할 수 있습니다. 이 경우 얇은 동축케이블은 두꺼운 동축케이블보다 저렴하며, T연결 탭보다 훨씬 저렴하기 때문에 10Base5보다 비용적면에서 더 효과적입니다. 얇은 동축 케이블은 매우 유연하기 때문에 설치도 간편합니다. 그러나 얇은 동축 케이블은 높은 레벨의 감쇠현상 때문에 각 세그먼트의 길이는 185m(200m가까이)를 초과할 수 없습니다.

 


10Base-T: Twisted-Pair Ethernet

10 Base-T는 물리적인 스타 토플로지를 사용합니다. 스테이션은 두 쌍의 트위스트 케이블을 통해 허브와 연결됩니다.

 

트위스트 케이블 두 쌍은 허브와 스테이션 사이에 두 경로(송수신)을 만들 수 있습니다.

모든 충돌은 이 허브에서 발생합니다. 10Base2 또는 10Base5와 비교하여 실제로 허브가 동축 케이블의 충돌대비까지 대체하는 것을 알 수 있습니다. 트위스트 케이블의 최대 길이는 트위스트 케이블의 감쇠현상을 최소화하기 위해서 100m로 정의되어 있습니다.

 

 

lOBase-F: Fiber Ethernet

 

다양한 유형의 광섬유 10Mbps 이더넷이 있지만, 가장 일반적인 것은 10BaseF입니다. 10BaseF는 허브에서 스테이션을 연결하는 스타형 토플로지를 사용합니다. 스테이션은 두 개의 광섬유 케이블을 이용하여 허브와 연결합니다.



13.3 CHANGES IN THE STANDARD

10 Mbps 표준 이더넷은 높은 데이터 전송률을 지원하기 많은 변화가 있었습니다. 실제로 이러한 변화를 통해 다른 고속 데이터 LAN과 호환되는 이더넷변화의 길을 열었습니다.

 

 

Bridged Ethernet

이더넷 발전의 첫 단계는 브리지로 인한 LAN에 대한 변화 부분이었습니다. 브리지는 이더넷 LAN에 대역폭 증가와 충돌 도메인에 관한 두 가지 효과를 가져왔습니다.

 

Raising the Bandwidth

브리지 결합이되지 않는 이더넷 네트워크에서는, 전체 용량(10Mbps)을 모든 스테이션과 전송 프레임이 공유합니다. (스테이션은 네트워크와 대역폭 공유). 이 경우 오직 한 개의 스테이션만 전송할 프레임이 있는 경우 전체 용량에는 이득이 될 수 있습니다. 그러나 두 개 이상의 스테이션이 네트워크를 사용하고 싶을 때 성능을 공유하게 될 것 입니다.

예를 들어 두 개의 스테이션이 많은 프레임을 보내야 할 경우 두 스테이션은 돌아가며 사용할 것입니다. 한 스테이션이 전송을 할 때 다른 스테이션은 전송을 보류할 것입니다. 이를 통해 각 스테이션은 평균 5Mbps 속도로 전송한다고 알 수 있습니다.

 

다시 말해서 브리지는 두개 이상으로 네트워크를 분할합니다. 각 네트워크에서 대역폭은 독립적입니다.

 

Separating Collision Domains

브리지의 다른 장점으로는 충돌 도메인을 분리 할 수 있다는 것입니다.

 

 

Switched Ethernet

스위치 이더넷에서의 대역폭은 스테이션과 스위치 사이에서만 분배됩니다. 또한 충돌 도메인은 N 도메인으로 나누어집니다. 브리지 이더넷을 바탕으로 변화한 스위치 이더넷은 더 빠른 이더넷을 더 빨리 볼 수 있도록 해 주었습니다.

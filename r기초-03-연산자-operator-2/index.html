03. 연산자 Operator

* 본 블로그의 모든 게시물은 전문적인 내용이 아님을 알려드립니다.

 * 내용에 오류가 있거나 불친절한 설명이 있는 경우 언제든지 댓글로 의견을 남겨주세요!


연산자에는 산술 연산자, 비교 연산자, 논리 연산자가 있으며 이에 대해 알아보겠습니다.

1. 산술 연산자

설명 

예

 *

 곱셈 

 a*b 

 

 

 

 
 

 /

 나눗셈 

 a/b 

 

 

 

 
 

 ^

 지수 승 

 a^2 

 

 

 

 
 

 %%

 나머지 

 a%%b 

 

 

 

 
 

 %/%

 몫 

 a%/%b 

 

 

 

 

산술 연산자는 대부분 익숙한 것들이라 가볍게 넘어가는 경우가 많으나 주의깊게 봐야 하는 연산자가 있습니다.

바로 %%, 나머지 연산자인데 이 연산자를 어떻게 사용하느냐에 따라 정말 많은 것을 할 수 있습니다.

예시는 다음에 포스팅할 indexing 에서 다루도록 하겠습니다.

2. 비교 연산자

– 비교 연산자는 특정 조건에 대해서 참 거짓으로 구분 짓게 해주며 결과 값을 TRUE와 FALSE로 반환합니다.

 

연산자 

설명 

예 

 

 

 

 
 

 ==

 같은가 

a==b

 

 

 

 

 

  !=

 다른가

a!=b 

 

 

 

 
 

 >

 크다 

a>b 

 

 

 

 
 

 >=

 크거나 같다 

a>=b 

 

 

 

 
 

 <

 작다

a<b 

 

 

 

 
 

 <=

 작거나 같다 

a<=b 

 

 

 

 

  조금 더 자세하게 예를 들어 봅시다.

> x <- 0.3

> y <- 0.5

> x < y

[1] TRUE


> x != y

[1] TRUE


> x == y

[1] FALSE



그런데 여기서 살짝 주의하셔야 할게 있습니다. x와 y에 각각 0.1, 0.3을 뺀 후 값이 서로 같은지를 비교해보겠습니다.

> x1 <- x – 0.1

> x1

[1] 0.2


> y1 <- y – 0.3

> y1

[1] 0.2


> x1 == y1

[1] FALSE


> x1 != y1

[1] TRUE

x에서 0.1을 뺐으니 0.2이고 y에서 0.3을 뺐으니 0.2인데 어째서 값이 다르다고 나올까요?
java에서도 확인해보니 서로 다른 값이라고 인식하는데 왜 이런 일이 일어날까요? 
‘이는 정밀도 때문입니다.
우리가 10진수로 숫자를 입력하면 컴퓨터는 2진수로 변환하여 계산합니다. 
정수일 경우에는 상관이 없지만 소수를 계산하게 되면 어떤 경우에는 컴퓨터가 2진수로 변환시 무한소수로 가게 되는 경우가 있어 100% 정확하게 표현할 수가 없습니다.
따라서 두 벡터 사이에 차이가 생겨 값이 같지 않다고 나오게 됩니다.’

출처: http://catkim.tistory.com/2?category=635308 [김굉의 알알]

라고 합니다. 이외에도 소수를 표현하는 타입이 여러개인 타 언어(ex java)에는 소수의 값이 타입에 따라 다른 경우도 있습니다.
float(.1) 0.100000001490116119384765625
double(.1) 0.10000000000000000555111512312578270211815834045 41015625

출처: https://randomascii.wordpress.com/2012/02/25/comparing-floating-point-numbers-2012-edition/


위 출처의 블로그 작성자는 이와 같은 문제를 해결하기 위해 *all.equal 을 사용합니다.

> all.equal(x1,y1)

[1] TRUE

즉 저희가 얻을 수 있는 결론은 비교 연산자를 통해서 값을 비교할 때 조건문에 소수가 들어간다면 R에서 출력해주는 논리값이 100%옳다고 말할 순 없겠다는 것입니다.

3. 논리 연산자

 

연산자 

설명 

예 

 

 

 

 
 

 !

 아니다 

!a

 

 

 

 

 

 |

 or

a|b 

 

 

 

 

 

 ||

 or (객체의 첫번째 요소만 비교) 

a||b 

 

 

 

 

 

 &

 and 

a&b 

 

 

 

 

 

 &&

 and (객체의 첫번째 요소만 비교)

a&&b 

 

 

 

 

순서대로 예시를 통해 살펴보겠습니다.

! 느낌표는 ~이 아니다를 뜻하며 !a는 참이 아니기 때문에 FALSE 를 출력하여 줍니다.

> a <- TRUE

> !a

[1] FALSE



다음은 OR연산자입니다. 사용법은 Shift를 누르고 엔터 위에있는 \키를 눌러 주시면 됩니다.

두개 모두가 FALSE 일때를 제외하곤 전부 참이 됩니다. 숫자로 해도 동일한 결과가 나옵니다.

> TRUE | TRUE

[1] TRUE


> TRUE | FALSE

[1] TRUE


> FALSE | TRUE
[1] TRUE

> FALSE | FALSE

[1] FALSE


> 3 | 2                            #0이 아닌 모든 숫자는 TRUE이다.

[1] TRUE

> 0 | 2

[1] TRUE

> 0 | 0

[1] FALSE


다음은 AND 연산자 입니다. 곱의 느낌과 유사하고 두개 모두 참인 경우만 참이 됩니다.

> TRUE & TRUE

[1] TRUE


> TRUE & FALSE

[1] FALSE


> FALSE & TRUE
[1] FALSE

> FALSE & FALSE

[1] FALSE


> 3 & 2 

[1] TRUE

> 3 & 0

[1] TRUE

> 0 & 0

[1] TRUE



다음은 AND와OR연산자가 두번씩 쓰여진 ||와 && 연산자 입니다. 

우선 위의 예시에 나온 내용들은 ||나 &&로 교체하여 사용하여도 같은 결과값을 반환합니다.

> 0 && 0

[1] TRUE


> TRUE || FALSE

[1] TRUE

&& 와 ||는 두 객체를 비교할 때 전부를 비교하는 것이 아닌 첫번째 요소만 비교를 합니다. 위의 예시론 설명이 불가해 다음 예를 들겠습니다.

> a <- c(1, 0, 2)                            # a와 b에 값을 할당, c( ) 로 감싸주는것에 대한 내용은 다음 포스트 vector(벡터)에서 설명하겠다.

> a

[1] 1 0 2


> b <- c(0, 1, 3)

> b

[1] 0 1 3


> a | b

[1] TRUE TRUE TRUE

> a & b

[1] FALSE FALSE TRUE

> a || b

[1] TRUE

> a && b

[1] FALSE

둘의 차이점은 | 와 & 연산자는 a와 b안의 모든 요소를 각각 비교하는 반면 ||와 && 연산자는 첫번째 요소값(a의 1과 b의 0)만 비교합니다.

이외에도 논리 연산자 &와 | 는 나중에 원하는 조건에 맞는 값을 추출할 때 많이 사용됩니다.
흔히 ~이상(초과) ~이하(미만), 예를 들어 1 <= x <= 5를 코드로 구현하기 위해선


 x > 1 & x < 5            #1보다 크고(and) 5보다 작은 x

라고 사용해야 됩니다.

4. 특수 연산자


연산자 중에서 위의 내용에 해당이 안되면서 %any%형식으로 되어있는 특이한 연산자들이 몇가지 있습니다.

그 중 행렬곱 %*%, 포함 여부를 묻는%in%, 마지막으로 연산자를 생성하는 방법에 대해서 알아보도록 하겠습니다.


4-1) 행렬곱 %*%


– 행렬의 곱은 일반 곱셈과 다르다. 이를 R에서는 a%*%b 와 같은 형식으로 사용한다.

– 피연산자 a와 b는 적어도 하나는 행렬 형태를 따라야 하며 기본적인 행렬 곱이 가능한 형태여야 한다.




4-2) 포함여부 %in%


– 데이터 전처리를 하면서 a가 b에 포함이 되어있는가를 따지는 것은 자주 사용된다.  a%in%b와 같이 사용한다.

> a <- 1:3

> b <- 2:4

> a %in% b

[1] FALSE TRUE TRUE

4-3) 연산자 생성 %” “%

– R에서는 연산자를 만들어 사용할 수 도 있다.

– 이름과 연산내용은 어느것이 들어오든 상관없다.

– 양식은 %”이름”% <- function(x,y) {연산 내용} 의 형태이다. 자승을 취해주는 연산자를 만들어 보자면 다음과 같다.

   

> %”s”% <- function(x,y) {                                #function에 대해서는 차후 게시할 예정

return(x^y)

}



> 2%s%3

[1] 8


* 사용 함수 정리


*01. all.equal()

–  R스크립트 창에 ?함수명 혹은 help(“함수명”)을 치면 우측 하단의 창에 함수에대한 설명이 나옵니다. ?함수명을 주로 사용합니다.

> ?all.equal

> help(“all.equal”)


– 다음은 실행 결과입니다


Test if Two Objects are (Nearly) Equal


Description

all.equal(x, y) is a utility to compare R objects x and y testing ‘near equality’. If they are different, comparison is still made to some extent, and a report of the differences is returned. Do not use all.equal directly in if expressions—either use isTRUE(all.equal(….)) or identical if appropriate.

(중략)

– 간략하게 해석을 해보자면 x와 y를 비교할 때 ‘near equality’ 근사하게 같은지를 비교합니다.

– 마지막 줄을 읽어 보면 if 표현식에 all.equal을 직접 사용하지 말고 *isTRUE() 혹은 *identical() 안에 사용하라고 나와 있습니다.

  모르는 함수가 나오더라도 당황하지 말고 항상 ?를 통해서 사용법을 찾는법에 익숙해 지셔야 합니다.

  ? 와 help로 해결이 되지 않는 문제는 구글링을 통해서 찾는 연습 또한 필요합니다.

> ?isTRUE                                        #?isTRUE시 isTRUE(x)는 identical(TRUE, x)의 축약형이라고 설명한다.

> ?identical


> isTRUE(all.equal(x1,y1) == TRUE)                #isTRUE(), 괄호안의 값이 참인지 판별, 

[1] TRUE

> identical(all.equal(x1,y1) , TRUE)                #all.equal(x1,y1)의 결과와 TRUE가 같은지 비교

[1] TRUE

– 보시면 isTRUE는 두 대상을 연산자를 통해 비교를 하고 identical은 ,(콤마)를 통해 비교하는 것을 알 수 있습니다.
